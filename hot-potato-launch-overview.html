Connected Services
Railway
Infra
Hosts the Node.js backend API. Handles souvenir generation triggers, IPFS uploads, and serves game state to the frontend.
railway.app · auto-deploys from GitHub main
Alchemy
RPC
Provides the Base blockchain RPC endpoint. All contract reads (game state, history) and backend writes (setSouvenirURI) go through Alchemy.
alchemy.com · key stored in Railway env → RPC_URL
Pinata
IPFS
Pins souvenir images and JSON metadata to IPFS. Each souvenir NFT's tokenURI points to a Pinata-hosted metadata file.
pinata.cloud · key stored in Railway env → PINATA_JWT
OpenAI
Image Gen
Generates unique souvenir artwork via DALL-E when a PotatoPassed event fires. Rarity tier and hold duration influence the prompt.
openai.com · key stored in Railway env → OPENAI_API_KEY

Security Concerns
Critical
Alchemy API Key in Git History
The old Alchemy RPC URL (containing key CCsT7yY4zuEqcoCPeivbS) was hardcoded in multiple backend files and is still present in git commit history, even though it was removed from the current codebase.
→ Rotate the key in Alchemy dashboard NOW. Then rewrite git history with BFG Repo Cleaner before any mainnet push.
Critical
Backend Wallet Private Key
WALLET_PRIVATE_KEY is stored as a Railway environment variable. If Railway is compromised or credentials leaked, an attacker can call setSouvenirURI with malicious metadata. The wallet also pays gas — if drained, souvenir generation silently fails.
→ Use a fresh wallet for mainnet with minimal ETH balance (top up as needed). Enable Railway 2FA. Never commit the key to git.
Warning
Creator Wallet Linked to Personal History
The current creator wallet (0x995a…) is derived from your existing MetaMask seed and has prior on-chain history (Coinbase deposit, OpenSea trades). This links the project wallet to your personal identity on-chain.
→ Create a fresh wallet from a new seed phrase and fund via CEX withdrawal for a clean on-chain separation. See Wallet Plan below.
Warning
Contract Not Verified on Basescan
The deployed contract source code is not yet publicly verified. Users and marketplaces cannot inspect the contract logic, which reduces trust and may prevent OpenSea royalty enforcement.
→ Verify contract source on Basescan before mainnet launch via Remix "Verify & Publish" or Hardhat verify plugin.
Info
No Frontend Rate Limiting on Backend API
The /api/souvenir/generate endpoint has no authentication or rate limiting. Anyone who knows the URL could spam souvenir generation requests, triggering OpenAI and Pinata API calls at your cost.
→ Add a shared secret header check on the generate endpoint, verified in Railway env vars. Low priority for testnet, important for mainnet.
Info
OpenAI & Pinata Keys Exposed if Railway Is Breached
All API keys live together in Railway environment variables. A breach of the Railway account exposes all service credentials simultaneously.
→ Enable Railway 2FA. Consider using Railway's secret references rather than plain env vars. Rotate all keys after mainnet launch.

Clean Wallet Plan (Before Mainnet)
1
Deployer Wallet
Deploys the mainnet contract. Becomes the contract owner. Receives the 5% creator fee on every sale. Keep this wallet secure and offline after deployment.
Fund via: CEX withdrawal (Coinbase → new address). ~0.01 ETH needed for deploy gas.
2
Backend Wallet
Used only by Railway to call setSouvenirURI. Needs a small ETH balance for gas. Private key lives in Railway env vars only.
Fund via: small transfer from Deployer wallet. Top up as needed. Keep balance minimal.
3
Creator Fee Wallet
Receives the 5% creator fee from every purchase. Can be the same as Deployer or a separate cold wallet. Set as CREATOR_WALLET in the contract at deploy time.
Keep separate from Backend wallet. Consider a hardware wallet (Ledger) for mainnet.
