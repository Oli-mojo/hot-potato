2. Where v2 needs to get us

The v2 story depends on three new capabilities that don't exist in v3:

  1. Two pools — live Pot + Last Supper Reserve. The live Pot pays the outgoing seller a small tier-slice on each sale. The Reserve accrues an equal amount per sale and only pays out at finalSeal(), tier-weighted to whoever is sitting. See v4 mechanic spec for locked numbers.
  2. Sprouting stages — floor-lifting, not decay. A stage index 0–5 that the contract advances based on elapsed hold time. Each stage raises the minimum next ask by a cumulative % (Fresh +0%, Leafing +10%, Sprouted +25%, Bloomed +50%, Harvest +100%, Eternal +200%). Emits MetadataUpdate (ERC-4906) so marketplaces refresh the artwork.
  3. Last Supper / game ending — state machine ACTIVE → LAST_SUPPER → SEALED. Last Supper triggered by 30 days without a transfer OR by spectator vote stake crossing ~10% of Reserve. 48-hour window, then anyone can call finalSeal(). Reserve pays the sitter 10%/25%/50%/75%/100% based on final tier; residual rolls to Season 2.

Spectator mechanics (rooting, glance, limit buys, souvenir votes) are on the roadmap but not in the P1 scope, except for the minimal voteLastSupper() hook — needed for the spectator-triggered endgame path. Full Hub ships as v4.1 / v4.2.

P1 scope for contract v4: Pot + Reserve accounting, sit-tier on-sale slice, sprouting stage machine with floor lift (no decay), Last Supper state transitions (timer + vote triggers), tier-weighted seal payout. Everything else (rooting pool, limit-order contract, glance fees, souvenir votes) is out of P1.

// ───── v4 ADDITIONS ────────────────────────────────────────

// State machine
enum GameState { ACTIVE, LAST_SUPPER, SEALED }
GameState public gameState;
uint256 public lastSupperStartedAt;
uint256 public lastTransferAt;                           // squat-killer clock
uint256 public constant LAST_SUPPER_WINDOW   = 48 hours;
uint256 public constant NO_TRANSFER_TIMEOUT  = 30 days;

// Two pools
uint256 public pot;                                      // live Pot — pays seller on each sale
uint256 public reserve;                                  // Last Supper Reserve — pays out at seal
uint256 public constant POT_SALE_BPS         = 50;       // 0.5% of sale to Pot
uint256 public constant RESERVE_SALE_BPS     = 50;       // 0.5% of sale to Reserve (equal split)
uint256 public constant POT_OVERPAY_BPS      = 2500;     // 25% of overpay to Pot
uint256 public constant RESERVE_OVERPAY_BPS  = 2500;     // 25% of overpay to Reserve

// Sit-tier on-sale slice (paid from Pot to outgoing seller)
// Cold 0% · Warm 0.5% · Hot 1.5% · Blazing 3.5% · Legendary 7.5%
uint16[5] public tierSaleSliceBps = [0, 50, 150, 350, 750];

// Last Supper final-seal share of Reserve (tier-weighted to current sitter)
// Cold 10% · Warm 25% · Hot 50% · Blazing 75% · Legendary 100% — residual rolls to Season 2
uint16[5] public tierFinalShareBps = [1000, 2500, 5000, 7500, 10000];

// Sprouting — each stage raises the minimum next ask (NOT decay)
enum Stage { FRESH, LEAFING, SPROUTED, BLOOMED, HARVEST, ETERNAL }
Stage public currentStage;
uint256 public stageEnteredAt;
string[6] public stageURIs;                              // IPFS URIs per stage

// Cumulative floor-lift on top of base min-increase (in BPS)
// Fresh +0% · Leafing +10% · Sprouted +25% · Bloomed +50% · Harvest +100% · Eternal +200%
uint16[6] public stageFloorLiftBps = [0, 1000, 2500, 5000, 10000, 20000];

// Stage advance thresholds (time since potato last changed hands)
uint256[6] public stageThresholds = [0, 3 days, 7 days, 14 days, 30 days, 60 days];

// Spectator vote-to-seal pool
mapping(address => uint256) public voteStake;
uint256 public totalVoteStake;

// Events
event PotFunded(uint256 amount, string source);     // source = "sale"|"overpay"|"glance"|"stake"
event ReserveFunded(uint256 amount, string source);
event PotPaidOut(address indexed to, uint256 amount, uint8 tier);
event ReservePaidOut(address indexed to, uint256 amount, uint8 tier, uint256 residual);
event StageChanged(uint8 oldStage, uint8 newStage);
event LastSupperBegan(uint256 timestamp, address holder, string trigger);
event VoteStaked(address indexed voter, uint256 amount, uint256 totalStake);
event GameSealed(uint256 timestamp, address eternalHolder, uint256 reservePaid, uint256 residualForSeason2);
event MetadataUpdate(uint256 _tokenId);             // ERC-4906

// Keeper-callable — anyone can poke the stage forward
function pokeStage() public {
    require(gameState == GameState.ACTIVE, "game over");
    uint256 held = block.timestamp - holderInfo.purchaseTimestamp;
    uint8 newStage = uint8(currentStage);
    while (newStage < 5 && held >= stageThresholds[newStage + 1]) {
        newStage++;
    }
    if (newStage != uint8(currentStage)) {
        emit StageChanged(uint8(currentStage), newStage);
        currentStage = Stage(newStage);
        stageEnteredAt = block.timestamp;
        emit MetadataUpdate(POTATO_TOKEN_ID);
    }
}

// Minimum next ask — tiered base increase, then floor-lifted by current stage
function minNextAsk() public view returns (uint256) {
    uint256 baseBps = _minIncreaseBps(currentPrice);                      // from v3: 25/15/10/7/5%
    uint256 liftBps = uint256(stageFloorLiftBps[uint8(currentStage)]);
    // combine multiplicatively: base * (1 + liftBps/10000)
    uint256 baseFloor = (currentPrice * baseBps) / 10000;
    return (baseFloor * (10000 + liftBps)) / 10000;
}

// Spectator path to trigger Last Supper early
function voteLastSupper() external payable {
    require(gameState == GameState.ACTIVE, "game over");
    require(msg.value > 0, "zero stake");
    voteStake[msg.sender] += msg.value;
    totalVoteStake += msg.value;
    emit VoteStaked(msg.sender, msg.value, totalVoteStake);

    // Threshold: 10% of current reserve, or 0.1 ETH floor if reserve is small
    uint256 threshold = reserve / 10 > 0.1 ether ? reserve / 10 : 0.1 ether;
    if (totalVoteStake >= threshold) {
        _beginLastSupper("spectator-vote");
    }
}

// Timer path — anyone can poke if 30 days with no transfer
function pokeLastSupperTimer() external {
    require(gameState == GameState.ACTIVE, "game over");
    require(block.timestamp >= lastTransferAt + NO_TRANSFER_TIMEOUT, "too soon");
    _beginLastSupper("no-transfer-timeout");
}

function _beginLastSupper(string memory trigger) internal {
    gameState = GameState.LAST_SUPPER;
    lastSupperStartedAt = block.timestamp;
    // The vote-stake pool folds into the reserve at transition
    reserve += totalVoteStake;
    emit ReserveFunded(totalVoteStake, "vote-pool");
    totalVoteStake = 0;
    emit LastSupperBegan(block.timestamp, ownerOf(POTATO_TOKEN_ID), trigger);
}

// Seal — callable by anyone after the 48h Last Supper window closes
function finalSeal() external nonReentrant {
    require(gameState == GameState.LAST_SUPPER, "not in last supper");
    require(block.timestamp >= lastSupperStartedAt + LAST_SUPPER_WINDOW, "window still open");
    gameState = GameState.SEALED;

    address sitter = ownerOf(POTATO_TOKEN_ID);
    uint8 finalTier = _currentSitTier();                   // 0..4
    uint256 shareBps = tierFinalShareBps[finalTier];
    uint256 sitterPayout = (reserve * shareBps) / 10000;
    uint256 residual = reserve - sitterPayout;             // rolls to Season 2 contract
    reserve = residual;                                    // leave residual on-chain for Season 2 transfer

    (bool ok,) = sitter.call{value: sitterPayout}("");
    require(ok, "payout failed");

    emit ReservePaidOut(sitter, sitterPayout, finalTier, residual);
    emit GameSealed(block.timestamp, sitter, sitterPayout, residual);
}

And modify buyPotato() to:

  1. Reject if gameState == SEALED. If LAST_SUPPER, still allow buys during the 48h window.
  2. Call pokeStage() at entry so every sale self-heals the stage (no reliance on external keeper).
  3. Use minNextAsk() as the price floor check — this is tiered base + sprout floor-lift, never below the current floor.
  4. Compute the outgoing seller's sit-tier from hold duration, look up tierSaleSliceBps[tier], carve that slice off the live Pot, and pay it to them on top of the 95% sale proceeds.
  5. Split Pot-funding across both pools: 0.5% of sale → pot, 0.5% of sale → reserve; 25% of overpay → pot, 25% of overpay → reserve. Emit PotFunded and ReserveFunded with sources.
  6. Update lastTransferAt = block.timestamp — resets the 30-day squat-killer clock.
  7. Reset currentStage = FRESH, stageEnteredAt = block.timestamp, emit StageChanged + MetadataUpdate.

No decay anywhere. effectivePrice() from earlier drafts is removed — the price a sniper must pay equals max(seller's ask, minNextAsk()), which is monotonically non-decreasing within a hold. Fortification, not erosion.

5. Phase C — Backend extended for v4

Goal: backend exposes new v4 data shapes to the frontend, and fixes the Railway persistence gap.

C1

BE Fix Railway persistence

The in-memory state issue (lost on redeploy) must be resolved before v4 data depends on it. Three options, pick one:

  1. Add a Postgres add-on on Railway. Cheap (~$5/mo), migration-friendly. Use prisma or knex. Persist rooter stakes, souvenir cache, event log.
  2. Go stateless. Every request rebuilds from on-chain reads + a public RPC. Slow but bulletproof. Fine for v4 launch since we don't expect huge traffic.
  3. Use a managed KV (Upstash Redis). Good middle ground if state is simple.

Recommendation: go with option 2 (stateless) for launch, add Postgres later when we need rooter bookkeeping.

9. Risks & watch-outs

Re-entrancy on Pot + Reserve payouts. The new buyPotato() and finalSeal() both send ETH externally. Both must be wrapped in nonReentrant, and both must follow CEI — update pot, reserve, gameState, and stage state BEFORE the external .call{value:}(). The malicious-seller attack vector is real; test it explicitly.

Pot-drain via sit-tier gaming. Because Legendary is reached at 30 days, an early holder can time the market and take 7.5% of a small Pot + 100% share of a small Reserve at finalSeal() despite barely contributing. Mitigation: require a minimum Pot/Reserve balance (e.g. 1 ETH combined) before the highest tier pays out, OR anneal tierSaleSliceBps during the first 30 days of a season, OR make the 30-day no-transfer trigger a min-Pot-size check as well.

Metadata URL fragility. If the backend goes down, the dynamic /api/metadata/potato URL breaks, and OpenSea shows a broken image. Mitigation: cache the stage metadata JSON to Pinata after every stage change, so there's always an IPFS fallback. Ship this in C5.

Keeper downtime. If the stage-advance keeper dies, the contract still works but artwork stays stale. Mitigation: pokeStage() is public — anyone can call it, including the snipe flow itself. Call it at the top of buyPotato() so every sale self-heals the stage.

Last Supper timing. 48h is the locked default — balances urgency with timezone fairness. If telemetry says it's too tight post-launch, expose LAST_SUPPER_WINDOW as a governance variable (not a constant) so you can retune without redeploying.

Legal/regulatory (US). A contract that pays users based on how long they hold an NFT has shades of an investment contract. This is not the same as a yield-bearing token — no passive return, you must actively take and hold the potato — but counsel should weigh in before mainnet. Base's L2 position helps but does not eliminate risk.
