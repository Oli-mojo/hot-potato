1. Why v2 — not an incremental patch

The current site is built around the v1 mental model: buy the potato, hold for rarity, generate a souvenir when sold. The mechanics we've added since — the Pot, sit-tier ladder, sprouting/decay, Last Supper endgame, rooters, glances, limit buys, souvenir votes — each shift the player's core incentive loop. Trying to layer those onto the current UI would end up as a cluttered dashboard where nothing reads as the headline.

The right move is to rebuild the hero and rules surfaces around the new headline — the Pot is the center of gravity — while keeping souvenir gallery, history, and leaderboard mostly intact. Call the existing file index_v1.html archival, develop index_v2.html in parallel, and cutover once the contract mechanics go live on testnet.

Design thesis: the Pot balance is the new price anchor. Current price tells you cost-to-enter. Pot tells you prize-to-win. Players scan for "is the pot juicy?" before asking "can I afford a snipe?" — that ordering has to be visually true on the page.

What changed and why:

  • Stats row flips from price-anchored to Pot-anchored. Pot is the first thing a spectator reads. Price moves down to the CTA.
  • The "rarity building" heat bar is gone. It maps to an old incentive. The sit-tier ladder is the replacement and it tells a story: "you're Warm, you get there at Hot, and Legendary is a real thing other players have reached."
  • Sprouting is surfaced up here, not buried. The countdown to next stage is the second-most-watched number (after Pot) because it triggers visible change.
  • CTA wording. "Snipe the throne" is more aligned with the throne metaphor than "Buy the potato." It hints at drama and implies a loser (the current holder).

4. The Pot widget

This is the single most important new component. It needs to be prominent, live-updating, and emotionally evocative — a number that's going up.

│                                                              │

│   Bob's payout if sniped now:     +0.115 ETH (3% slice)      │
│                                                              │
│   30-day growth  ▁▂▂▃▃▄▅▅▆▆▇▇▇█                              │
│                                                              │
│   · 0.008 ETH added 14min ago  — Alice → Bob sale           │
│   · 0.021 ETH added 2h ago     — Dana → Alice sale          │
│   · 0.004 ETH added 6h ago     — rooter stake               │
│                                                              │
│   [ Root on Bob → earn if he survives ]                      │
└──────────────────────────────────────────────────────────────┘

Data model / implementation notes

  • pot.balance() — on-chain view. Poll every 15s or refresh on PotatoSold, RooterStaked, PotDistributed events.
  • currentHolder.payoutSlice — computed client-side from holdDuration → tier → potSlicePct[tier].
  • Sparkline: 30-day daily closes from indexer. Just a tiny SVG, no chart library needed.
  • "Last events" feed: the indexer's last 5 Pot-affecting events, live-streamed via SSE or a 15s poll.

Stage-change animation

When pokeStage() is keeper-called and MetadataUpdate(tokenId) fires, the site should:

  1. Play a 2-second crossfade on the hero image from old stage → new stage.
  2. Flash the "🌱 Leafing" chip into "🌿 Sprouted" and pulse gold for 3 seconds.
  3. Post a Discord/X bot message with the before/after image.
  4. Add a system message to the Pot events feed: "Potato sprouted to Sprouted stage — decay now −1.0% / day".

Community picks one trait for the next souvenir mint (color, accessory, pose). Low stakes, high engagement. Weekly cadence, weighted by past activity (snipes, roots, held-seconds).

8. Last Supper conditional UI

When GameState transitions from ACTIVE → LAST_SUPPER, the whole site has to feel different. This is the climax. Players should reach for their phones when they see it.

GameState transitions

[ ACTIVE ] ──(treasury depleted + no snipe in N days)──▶ [ LAST_SUPPER ]
[ LAST_SUPPER ] ──(final window expires)──▶ [ SEALED ]
↑ site palette shifts at each edge

Visual changes in LAST_SUPPER state

  • Palette shifts from warm-gold to deep-red / ember. Add an animated ember-glow behind the hero image.
  • Hero eyebrow becomes a giant jackpot countdown: "⏳ FINAL WINDOW CLOSES IN 23h 04m 11s — POT: 4.12 ETH — WINNER TAKES ALL."
  • Nav gets a pulsing "◉ LAST SUPPER" indicator.
  • Every new snipe triggers a full-screen takeover animation: "Dana has taken the throne — 4.12 ETH at stake."
  • Share-card generator button appears in hero: generates a PNG suitable for X with current holder + pot + countdown.
  • Rooting pool closes. UI greys out that component with "Rooting closed for the Last Supper."

SEALED state

Once finalized, the whole site becomes a permanent monument. Hero shows the Eternal Holder, the final pot (100% to them), and a "Savior NFT" claim flow if applicable. Spectator hub becomes read-only archives. The site is still useful — it's a shrine.

Engineering note: the LAST_SUPPER palette switch should be driven by a single CSS root-variable flip based on the on-chain gameState enum. Don't fork the codebase — just add body[data-state="last-supper"] selectors.

11. Backend & indexer changes

Event indexer

Listen for and persist these new events (all on-chain, triggered by the contract):

event PotatoSold(address from, address to, uint256 price, uint8 holdTier, uint256 potPayout);
event StageChanged(uint8 oldStage, uint8 newStage);
event RooterStaked(address rooter, address holder, uint256 amount);
event RooterSettled(address rooter, uint256 payout, bool holderSurvived);
event LimitOrderPlaced(address buyer, uint256 target, uint256 deposit);
event LimitOrderFilled(address buyer, address keeper, uint256 price);
event GlanceSent(address spectator, uint256 fee);
event SouvenirMinted(uint256 tokenId, uint8 parentStage);
event LastSupperBegan(uint256 timestamp, address holder);
event GameSealed(uint256 timestamp, address eternalHolder, uint256 finalPot);
event MetadataUpdate(uint256 tokenId);  // ERC-4906

Metadata endpoint

The existing /metadata/:id route has to return the correct image URI based on currentStage. When MetadataUpdate fires, invalidate the OpenSea cache via their refresh API, so stage changes propagate visibly on external marketplaces too.

Discord / X bots

New triggers:

  • Throne change → post before/after holder + price + Pot slice earned
  • Sprout event → post before/after potato image
  • Rooter settled → post top 3 winners
  • Last Supper began → pinned announcement, countdown sticker
  • Game sealed → final results thread with Eternal Holder

Persistence

The Railway in-memory state issue that bit us on pendingBoosts has to be fully resolved before v2. All game state lives on-chain; the backend is a cache and an event bus, never a source of truth. If the backend falls over, the site should still render from direct RPC reads (degraded, but functional).

Rough sizing: P1 ~1 week, P2 ~2-3 days, P3 ~2 weeks (depends heavily on contract iteration), P4 ~1 week. Total ~5 weeks front-end work assuming you're the single dev, or ~3 weeks with a second person.

13. Open questions

  1. Pot widget placement — above or beside the potato image? Above reads more dramatic, beside is more dashboard-like. Lean: above on mobile, beside on desktop.
  2. Does the sit-tier ladder replace the heat bar entirely, or sit alongside it? Lean: replace. The heat bar communicated the same axis (time held) but attached to a now-retired concept (rarity roll). Keeping both would confuse.
  3. Glance fee — tiny and to the Pot, or tiny and to the holder? To the Pot makes it a clean spectator-funds-the-game mechanic. To the holder gives it a "you're watching me, pay up" flavor that's also fun. Both work; pick one and stick with it.
  4. Rooter pool funded from Pot or separate? Spec says separate pool. The UI here assumes that. If separate, need a clear label so it's not confused with the main Pot.
  5. Do limit buys count as snipes for leaderboard purposes? Lean yes — the keeper is just an executor, the buyer's wallet is the one taking the throne.
  6. Should the share-card generator work in ACTIVE state too, or only during Last Supper? Probably all states but with a different template per state. More viral potential than you'd think.

Bottom line: this is a v2 redesign, not a patch. The hero and rules surfaces need full rebuilds; souvenir gallery, history, and leaderboard extend gracefully. Ship P1 (hero + Pot + living NFT) first — it carries 70% of the new story by itself. The rest compounds from there.
